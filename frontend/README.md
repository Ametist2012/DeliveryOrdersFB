# Delivery Orders — Frontend

Frontend часть приложения **Delivery Orders** — клиентское веб-приложение для управления заказами доставки.

Приложение предоставляет пользователям возможность регистрироваться, выполнять авторизацию через JWT, создавать и просматривать заказы доставки, а также использовать функциональность, зависящую от роли пользователя.

Frontend разработан с использованием React и TypeScript и взаимодействует с ASP.NET Core Web API через REST API.

---

## Возможности приложения

### Аутентификация

Приложение поддерживает:

- регистрацию пользователей;
- авторизацию через JWT;
- сохранение состояния пользователя;
- проверку авторизации;
- обработку ошибок входа и регистрации;
- автоматический выход пользователя при истечении действия токена.

После успешной авторизации frontend получает JWT токен:

```
POST /api/auth/login
```

Пример ответа:

```json
{
  "token": "jwt_token"
}
```

Токен используется для обращения к защищённым API:

```http
Authorization: Bearer YOUR_TOKEN
```

---

## Работа с заказами

Frontend позволяет:
- создавать новые заказы доставки;
- просматривать список заказов;
- открывать детальную информацию о заказе;
- получать заказ по номеру заказа;
- выполнять сортировку;
- использовать пагинацию;
- отображать данные отправителя и получателя;
- отображать параметры груза;
- удалять заказы для пользователей с ролью Admin.

Получение деталей заказа выполняется через:

```
GET /api/orders/{orderNumber}
```

---

## Пользовательские роли

В приложении реализована ролевая модель.

Доступные роли:

- User;
- Admin.

В зависимости от роли пользователя изменяется доступный функционал.

### User

Пользователь может:
- зарегистрироваться;
- авторизоваться;
- создавать заказы;
- просматривать доступные заказы;
- просматривать информацию о заказах.

### Admin

Администратор дополнительно может:
- просматривать список пользователей;
- управлять пользователями;
- удалять заказы;
- использовать административный раздел.

Перейдите в папку frontend:

```bash
cd frontend
```

```bash

```

# Запуск приложения

Запуск frontend в режиме разработки:

```bash
npm run dev
```

После запуска приложение будет доступно:

```
http://localhost:5173
```

Изменения исходного кода автоматически применяются благодаря:

```
Vite Hot Module Replacement (HMR)
```

# Backend

Frontend работает совместно с ASP.NET Core Web API.

Перед запуском frontend необходимо запустить backend:

```bash
cd backend

docker compose up -d

dotnet ef database update

dotnet run
```

Backend API:

```
http://localhost:5056
```

Swagger документация:

```
http://localhost:5056/swagger
```

---

# API

Frontend взаимодействует с backend через REST API.

## Authentication

| Метод | Endpoint | Описание |
|---|---|---|
| POST | `/api/auth/register` | Регистрация пользователя |
| POST | `/api/auth/login` | Авторизация пользователя |

---

## Orders

| Метод | Endpoint | Доступ |
|---|---|---|
| GET | `/api/orders` | User, Admin |
| GET | `/api/orders/{orderNumber}` | User, Admin |
| POST | `/api/orders` | User, Admin |
| DELETE | `/api/orders/{orderNumber}` | Admin |

---

## Admin

| Метод | Endpoint | Доступ |
|---|---|---|
| GET | `/api/admin/users` | Admin |
| POST | `/api/admin/users` | Admin |
| DELETE | `/api/admin/users/{id}` | Admin |

---

# Валидация

Frontend реализует клиентскую и серверную обработку ошибок.

Поддерживается:

- проверка обязательных полей;
- проверка форматов данных;
- проверка диапазонов значений;
- отображение ошибок API;
- обработка ошибок авторизации.

Пример ответа backend при ошибке:

```json
{
  "title": "One or more validation errors occurred.",
  "status": 400,
  "errors": {
    "CargoWeight": [
      "The Cargo's weight must be between 0.01 and 100000 kg"
    ]
  }
}
```

Ошибки автоматически отображаются пользователю в интерфейсе.
